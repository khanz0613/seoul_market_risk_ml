2. Seoul Market Risk ML System(하이브리드 ML 모델 기반 금융 서비스) 

기간: 2025년 8월 ~ 9월 (완료)

역할: ML 엔지니어/백엔드 개발자 (개인 프로젝트)

프로젝트 유형: 머신러닝 + 금융공학 하이브리드 모델 기반 위험 분석 시스템



프로젝트 개요

  • 하이브리드 ML 모델: Altman Z'-Score + Random Forest 결합으로 예측 정확도 향상
  • 구체적 대출 시뮬레이션: "1,140만원 대출로 위험도 28.3→60.0점 개선" 등 정량적 예측
  • 실제 매출 데이터: 서울시 6년간(2019-2024) 상권 데이터 ~5만 사업자 분석
  • 지출 추정 모델(75.45%) + 시계열 분석으로 재무 피처 자동 생성


기술 스택

Machine Learning & Data Processing

  • Python 3.11+
  • Pandas, NumPy
  • Scikit-learn (Random Forest, Logistic Regression)
  • Matplotlib, Seaborn (시각화)

금융공학 & 분석

  • Altman Z'-Score: 세계적으로 검증된 부도 예측 모델 구현
  • 하이브리드 피처: 재무건전성(40%) + 영업안정성(45%) + 업종비교(15%)
  • 시계열 분석: 매출 트렌드, 변동성, 성장성 정량화
  • 대출 효과 시뮬레이션: 대출 후 위험도 개선 예측

데이터 소스

  • 서울열린데이터광장 상권분석서비스(2019-2024)
  • 소상공인 평균 지출률 75.45% 기반 재무제표 추정



핵심 구현 사항

1. 하이브리드 ML 모델 구현

  • 재무건전성 (40%): Altman Z'-Score 기반 부도 예측 모델
  • 영업안정성 (45%): 매출 성장률, 변동성, 지속성 분석
  • 업종비교 (15%): 업종 내 상대적 성과 평가
  • Random Forest: 최종 위험도 점수 예측 및 신뢰도 측정

2. 구체적 대출 시뮬레이션 시스템

  • 위험 사업자: "11,400만원 대출로 28.3점→60.0점 개선"
  • 우수 사업자: "78.5점 양호 상태, 성장 투자 검토"
  • 월 상환액, 위험도 개선 효과, 예측 신뢰도 제공

3. 성과 및 차별점

  • 학습 데이터: 서울시 ~50,000개 사업자 (2019-2024)
  • 예상 성능: 회귀 R² 0.85+, 분류 정확도 90%+
  • ❌ 임의 가중치 → ✅ 검증된 Altman Z'-Score + ML
  • ❌ "대출 검토 권장" → ✅ "1,140만원 대출로 점수 개선"


주요 기여사항

  • 하이브리드 ML 모델 설계: Altman Z'-Score + Random Forest 결합 아키텍처 설계
  • 데이터 전처리 파이프라인: 서울시 6년간 상권 데이터 클리닝 및 피처 엔지니어링
  • 지출 추정 모듈: 소상공인 평균 지출률 75.45% 기반 재무제표 자동 생성
  • 대출 효과 시뮬레이터: 구체적 대출 금액별 위험도 개선 효과 예측
  • 실시간 예측 시스템: 사업자 데이터 입력 시 즉시 위험도 산정 및 맞춤 추천



3. Fan.zip - 인플루언서 팬 종합 서비스 플랫폼

기간: 2025년 상반기 ~ 하반기

역할: 백엔드 개발자 + DevOps 엔지니어 + ERD 총괄(팀 프로젝트)

프로젝트 유형: 대규모 상용 서비스 플랫폼

GitHub Repository: https://github.com/fanzip/fanzip-server



프로젝트 개요

인플루언서와 팬을 위한 종합 서비스 플랫폼으로, 팬과 인플루언서 간의 소통을 돕고 팬 문화 생태계를 확장하는 것이 목표


기술 스택

Backend

  • Java + Spring Framework
  • Spring Security
  • MyBatis
  • MySQL
  • Redis (캐싱/세션 관리)
  • OAuth 2.0 + JWT (인증/인가)

Frontend

  • Vue.js 3 (Composition API)
  • Pinia (상태 관리)
  • Vue Router
  • Vite (빌드 도구)
  • Tailwind CSS

Infrastructure & DevOps

  • AWS EC2, RDS, S3 (클라우드 인프라)
  • Docker (컨테이너화)
  • Nginx (리버스 프록시, 로드 밸런서)
  • SSL/HTTPS (보안 통신)
  • GitHub Actions (CI/CD 파이프라인)
  • Firebase Cloud Messaging (FCM) - Push 알림

Monitoring & Analytics

  • Prometheus (시스템 CPU/메모리, API 응답 시간 모니터링)
  • Grafana (대시보드 시각화)
  • Alert Manager (Slack 연동 자동 알림)
  • GPT-4 API (AI 서비스 리포트 생성)



주요 구현 기능

  • 멤버십 시스템: VIP, GOLD 등급 회원 구독/청취 가능
  • 디지털 팬카드: 개인 인플루언서의 카드를 수집, 개인 및 개별 응원 메시지
  • 팬미팅 예약: 실시간 QR 코드 생성 시스템
  • 교환 메시지 서비스: 팬과 인플루언서 간 소통
  • 마이페이지: 프로필 및 개인 응원 히스토리
  • AI 리포트: 팬 문화 데이터 기반의 AI 서비스 리포트 생성



주요 기여사항 (백엔드 + DevOps)

  • 대규모 서비스 아키텍처 설계: 사용자 10만 명 이상 처리 가능한 확장성 있는 시스템 설계
  • ERD 설계: 전체 플로우를 기반으로 철저한 분석을 통해 ERD 전체 구축
  • 동시성 제어: Redis 기반 분산 락(Distributed Lock)을 도입하여 동시성 문제 해결
  • 실시간 QR 시스템: 30초마다 갱신되는 동적 QR 코드 + FCM 푸시 알림 연동
  • CI/CD 자동화: GitHub Actions를 통한 배포 파이프라인 구축
  • AWS 클라우드 인프라: EC2, RDS, S3를 활용한 확장 가능 인프라 구축
  • Docker + Nginx 운영: 컨테이너화된 애플리케이션 + 리버스 프록시 + SSL/HTTPS 적용
  • 모니터링 시스템: Prometheus + Grafana + Alert Manager로 실시간 시스템 상태 모니터링



시스템 아키텍처

Vue.js 3 Frontend ↔ Nginx (Load Balancer + SSL/HTTPS)

↓

Spring Boot API (Docker Container)

↓

AWS RDS (MySQL) + Redis Cache

↓

GitHub Actions CI/CD → AWS EC2 Deployment

↓

Prometheus + Grafana + Alert Manager Monitoring

프로젝트 개요

실시간 채소 가격 정보와 가격 기반 맞춤형 레시피 추천 서비스

  • Data Analysis: Python, Pandas
  • Web Scraping: Selenium, Requests
  • Backend: Spring Boot, JPA
  • Database: MySQL
  • Frontend: Flutter



주요 기여사항

  • 데이터 수집 시스템: 농산물 가격 정보 크롤링 기능 구현
  • 데이터 수집 자동화 인프라 구축: AWS Lambda와 AWS EventBridge를 이용해 매일 같은 시간에 당일의 농산물 가격을 크롤링 하도록 구현

기술 트러블슈팅 경험

배포 및 인프라 관련

1. GitHub Actions CI/CD 파이프라인 구축

  • 문제상황: 수동 배포로 인한 휴먼 에러 및 배포 시간 지연
  • 해결방안: GitHub Actions를 활용한 자동화된 CI/CD 파이프라인 구축
  • 성과: 배포 시간 80% 단축, 배포 오류 제로화


2. Nginx 리버스 프록시 및 JWT 설정 트러블슈팅

  • 문제상황: CORS 에러 및 JWT 토큰 인증 실패 이슈
  • 해결방안: Nginx 설정 최적화 및 JWT 미들웨어 구현
  • 성과: 프론트엔드-백엔드 통신 안정성 99.9% 달성


3. Redis 연동 및 Docker Compose 배포

  • 문제상황: 세션 관리 및 캐싱 성능 이슈
  • 해결방안: Redis 클러스터 구성 및 Docker Compose 기반 오케스트레이션
  • 성과: 응답 속도 60% 향상, 동시 접속자 처리 능력 10배 증가



API 개발 관련

1. Spring Legacy API 개발 트러블슈팅

  • 문제상황: 레거시 시스템 연동 및 성능 병목 현상
  • 해결방안: 비동기 처리 및 connection pool 최적화
  • 성과: API 응답 속도 70% 향상


2. Fancard API 코드 리뷰 및 개선

  • 문제상황: 코드 가독성 저하 및 중복 로직 증가
  • 해결방안: 리팩토링 및 디자인 패턴 적용
  • 성과: 코드 복잡성 50% 감소, 유지보수성 대폭 향상
